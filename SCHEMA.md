# 데이터 스키마 — 실제 공고문 5종 분석 기반

출처: 행복주택(대구)·국민임대(시흥능곡)·통합공공임대(횡성우천2)·영구임대(광주두암2)·50년공공임대(동해천곡5) 공고문 정독.

핵심 원칙: **3층 구조**
1. **UserProfile** — 사용자가 1회 입력하는 본인 정보 (유형 무관 공통 + 상황별)
2. **NoticeRequirements** — 공고별로 추출하는 요건/속성 (순위·배점 규칙을 구조화해 보관)
3. **Matching** — UserProfile를 각 NoticeRequirements에 대입 → 자격여부/순위/예상배점 계산

---

## 1. UserProfile (사용자 입력값)

### A. 핵심 (거의 모든 유형이 사용 — 필수 입력)
| 필드 | 타입 | 파생/용도 |
|---|---|---|
| 생년월일 | date | 나이 → 청년(19~39)·고령(65+)·성년 판정 |
| 무주택여부 | bool | 전 유형 필수 게이트 (분양권·입주권 포함) |
| 세대원수 | int | 소득·자산 기준표 적용 단위 (태아 포함) |
| 월평균소득(세대합산·세전) | int | 소득기준 % 충족 판정 |
| 총자산(세대합산) | int | 자산상한 판정 |
| 자동차가액(최고가 차량) | int | 자동차상한 판정 (총자산과 별도) |
| 거주지 시·군·구 | code | 거주지 순위 결정 |
| 해당지역 연속거주기간 | months | 순위/배점 |

### B. 가구·자녀·혼인 (행복·통합·국민 배점/자격)
| 필드 | 타입 | 파생 |
|---|---|---|
| 혼인상태 | enum(미혼/혼인중/예비/한부모) | 신혼·청년·한부모 자격 |
| 혼인신고일 | date | 혼인기간(7·10년) + 무주택기간 기점(만30세전 혼인 시) |
| 맞벌이여부 | bool | 신혼/통합 소득기준 +30%p 우대 |
| 자녀 목록[생년월일, 태아여부] | date[] | **여기서 전부 파생**: 자녀수·미성년수·만6세이하·2세미만(우선배정30%)·24개월이하·출산자녀수(’23.3.28이후, 가산용) |

### C. 청약저축 (국민임대 50㎡↑, 50년공임, 행복 자격요건)
| 필드 | 타입 | 용도 |
|---|---|---|
| 청약저축 가입여부 | bool | 순위 자격 |
| 가입기간 | months | 1순위(6개월/12개월 경과) |
| 납입인정횟수 | int | 순위(24회/6회) + 배점 |
| 저축총액(납입인정금액) | int | 40㎡초과 동순위 가점 |
| 무주택기간 | months | 50년공임 동순위 가점(3년 기준) |

### D. 자격·신분 (영구임대, 통합 우선공급 — multi-select)
| 필드 | 값 |
|---|---|
| 수급자 | 생계/의료/주거급여/차상위/해당없음 |
| 특수자격(복수) | 국가유공·5·18·참전·위안부·북한이탈·장애인(정도)·아동복지퇴소·노부모부양·다자녀·재해이재민 등 |
| 공급계층 자기선택(행복) | 대학생/청년/사회초년생/취업준비생/신혼/예비신혼/한부모/고령자/주거급여/산단근로자 |
| (산단·일자리연계) 직업·근속 | 근로/사업소득 유무, 근속개월, 근무지 시군구 |

> **입력 UX 권장:** A는 필수, B~D는 "해당되면 입력". 자녀는 생년월일만 받고 모든 자녀 관련 조건은 자동 파생. 소득/자산 %는 사용자가 계산할 필요 없이 시스템이 기준표로 환산.

## 3. Matching (사용자 ↔ 공고) — `match.mjs` 구현 (결정론·LLM 미사용)
`node match.mjs` → `profile.json`(UserProfile) × 110건 requirements.json → 산출:
1. **자격 게이트**: 무주택 / 소득(가구원수별 표 조회) / 자산상한 / 자동차상한 / 청약 / **계층(영구=수급자·고령자=65+·행복=청년/신혼)**. 하나라도 fail→**지원불가**, check 있으면→**확인필요**, 전부 pass→**지원가능**.
2. **거주지 순위**: 순위규칙 조건이 자유텍스트라 시군구/시도 **부분문자열 매칭**으로 최소 순위 산출(미명시면 최후순위 추정).
3. **예상 배점(추정)**: 배점표 항목명 키워드(거주기간·자녀수·청약납입·무주택기간)만 매핑해 합산, 못 읽는 항목은 `미반영`으로 투명 표시.
4. **면적/지역 필터**: 공급형·주택목록 전용면적 vs 희망면적 겹침 / 공고지역·주택목록 지역분포 vs 희망지역.
→ 결과: 판정+사유, 순위, 예상점(추정), 면적·지역 매칭, D-day. 정렬=판정→희망지역→D-day. 전체는 `data/match-result.json`.

> **핵심 원칙(정직성)**: 자유텍스트라 확정 못 하는 건 추측하지 않고 **"확인필요"+원문확인**으로 표시. 예) 영구임대/고령자/청약회차/자격증명. 옵션: `--all`(마감포함) `--possible`(지원가능만) `--region-only`(희망지역만) `--type=` `--profile=`.

### 공급형태 — 지원금형 vs 실물공급 구별 (결정론)
"전세보증금만 지원하고 집은 본인이 구하는 정책"과 "입주할 집(호실)이 특정된 공고"는 사용자 입장에서 완전히 다르다. `match.mjs`가 기존 필드만으로 구별한다:
- **지원형**: `전세지원 있음 && 주택목록 없음` → 청년전세임대1순위·신혼/신생아전세임대·다자녀전세임대 (한도 내 본인 탐색).
- **실물공급**: `주택목록 있음`(매입주택목록서 선택, 든든전세 포함) / `매입주택 있음`(목록 미연동) / 건설단지 호실(공급형 실면적).
- ⚠️ 든든전세는 "전세형"이지만 주택목록이 있으므로 **실물**로 분류된다(전세지원+주택목록 둘 다 → 주택목록 우선).
- 필터: `--supply=지원형|실물`, `--conversion`(분양전환형만). 출력에 〔지원형〕〔분양전환〕 뱃지.

## 4. 매입임대·전세임대 변형 (든든전세/청년전세임대 분석 반영)

건설임대와 구조가 달라 스키마에 아래를 **추가**한다.

### NoticeRequirements 추가 필드
```jsonc
{
  "상품구조": "건설임대|매입임대|전세임대",      // 자격·주택 모델 분기
  "모집유형": "정기|수시",                       // 수시=마감일 약함, 상시접수/물량소진시중단
  "분양전환": "분양전환형|비분양전환형|해당없음",  // 든든전세
  "자격요건": {
    "소득기준": "도시근로자%|중위소득%|없음|자격증명갈음", // 든든전세=없음, 청년전세1순위=자격증명갈음
    "자산상한": null,                            // 없을 수 있음
    "청약요건": "없음"
  },
  // 전세임대 전용
  "전세지원": {
    "지원한도": { "수도권": {"1인":120000000,"2인":150000000,"3인+":200000000}, "광역시": {...}, "기타": {...} },
    "한도초과허용": "1인 150% / 2인+ 200%",
    "본인부담": "보증금 100만 + 월세(지원금×금리/12)", "금리": "1.2~2.2%(규모별)",
    "대상주택조건": { "전용면적상한": 85, "부채비율": "90% 이하", "주택유형": ["단독","다가구","연립","아파트","주거용오피스텔"] },
    "지역범위": "전국", "거주지제한": "대학생=대학소재 시도+연접"
  },
  // 든든전세 전용
  "매입주택": { "임대보증금범위": [250000000,330000000], "월임대료": 0, "주택목록파일": "...xlsx",
                "권역": ["서울","인천","경기"], "면적규모선택": ["60㎡미만","60~70","70~80","80㎡+"],
                "임대기간": "2년, 재계약3회(최장8년)" }

```

### 선정방식 변형
- **든든전세**: 순위 없음. 배점 = 신생아 2점 + 자녀수(1~3점), 최대 5점 → 동점 추첨 → 동·호 무작위.
  - 신생아 = **공고일 기준 2년 이내** 출생·입양·태아 (≈ 24개월).
- **청년전세임대(1순위)**: 동순위 배점 없음. 자격(무주택+미혼+청년+1순위유형) 충족 시 물량 내 선정.

### UserProfile 추가 입력값
| 필드 | 용도 |
|---|---|
| 자격유형(수급자/한부모/차상위/해당없음) | 전세임대 1순위, 영구임대 자격 |
| 학적상태(재학/휴학/복학예정/졸업2년내) | 대학생·취준생 자격 |
| 대학 소재 시·도 | 청년전세임대 대학생 지역제한 |
| 입주 인원(1인/셰어2·3인) | 전세임대 지원한도 결정 |
| 희망 권역/지역 | 든든전세 권역, 전세임대 한도 |
| 희망 전세보증금 | 전세임대 본인부담·한도초과 계산 |

> **핵심 일반화**: 자격 게이트는 유형별로 (1)소득·자산 심사형(국민·통합·행복) (2)무주택+청약형(50년공임) (3)무주택 단일형(든든전세) (4)자격증명 갈음형(영구임대·청년전세 1순위)로 4갈래. 스키마의 `자격요건.소득기준`이 이 분기를 흡수한다.

### 정규화 규칙
- **선정방식**: 반드시 enum 1개. 복합이면 "혼합" + `선정방식상세`에 서술.
- **소득기준**: 무조건 object. 소득 없는 유형은 `{"종류":"없음", 나머지 null}`. **자격완화로 소득 배제 시** `종류:"없음"` + 비고에 근거(매처가 비고의 "소득요건/심사 배제·제한 없음"도 보정 인식).
- **계층별**: **캐논 객체**로 고정(추출 자유형 → `normalize-requirements.mjs`가 표준화). 키 enum = `대학생·청년·신혼·한부모·고령자·주거급여수급자·산업단지근로자·주거약자·일반`. 내부 필드 enum = `자산상한·자동차상한·소득기준·청약요건·연령·무주택·대상·비고`. 금액은 원(정수). 매처 `tierLimit`이 top-level 미기재 시 본인 계층값으로 평가하므로 **키/필드명 일관성이 매칭 정확도에 직결**.
- **임대료**: 단일이든 계층별이든 항상 배열(`임대료[]`). 통합공공임대 소득구간은 `구분:"N구간"`.
- **세대수**: 단지 `건설세대수` vs 공급형 `금회모집호수` 분리.
- **마감일/상태/접수시작 추가** → 수집기 meta에도 추가 필요(현재 미저장). 접수중 필터·D-day 알림의 핵심.
- **코드 로직 필요 항목**(육아지원 2세대 동시당첨 등)은 배점표 `비고`에 표시 → Matching 단계에서 처리.

> ⚠️ 수집기 보강 필요: 현재 meta.json에 마감일·상태·접수기간이 없음. 목록 HTML에 다 있으니 `lh-collect.mjs` 행 파싱에 추가해야 접수중 필터링 가능.

## 6. 분양 변형 (청약홈 OpenAPI 소스) — APT 분양정보 기반

청약 가점제는 임대 배점보다 표준·결정론적이라 소스만 있으면 계산이 깔끔하다.
| 항목 | 만점 | 산식 |
|---|---|---|
| 무주택기간 | 32 | 만30세(이전 혼인 시 **혼인신고일**)부터 산정. 1년미만 2점, 이후 1년당 +2점, **15년↑ 32점** |
| 부양가족수 | 35 | 0명 5점, 1명당 +5점, **6명↑ 35점** (배우자 + 직계존속〔3년↑ 동일등본 부양〕 + 미혼 직계비속〔만30세미만/동일등본〕) |
| 청약통장 가입기간 | 17 | 6개월미만 1점, 6~12개월 2점, 이후 1년당 +1점, **15년↑ 17점** |

### 6-3b. 신혼희망타운(공공분양) 표준 자격 — 결정론 인코딩 (`evalNHT`)
공공분양은 일반적으로 소득·자산 컷이 갭(공고문)이지만, **신혼희망타운은 표준 정책값**이라 단정한다(공고명 `/신혼희망타운/` 감지).
| 게이트 | 기준(연도 라벨) |
|---|---|
| 대상계층 | 혼인 7년 이내 · 만7세 미만 자녀 · 예비신혼(1년내) · 한부모(7세미만 자녀) |
| 무주택 | 무주택세대구성원 |
| 청약통장 | 주택청약종합저축 가입 6개월 경과 · 6회 이상 |
| 총자산 | 3억 6,200만원 이하 (**2026 적용**) |
| 소득 | 도시근로자 월평균 **130%**(맞벌이 **140%**) 이하 — 가구원수별(2024적용 도시근로자 100% 표 × rate) |
- 통과→**지원가능**, 명확한 미달→**지원불가**(추측 아님, 공시 정책값). 메시지에 연도·"공고문과 다를 수 있음" 명시.
- **본청약 caveat**(참고, 판정 안 깎음): 사전청약 당첨자 우선공급 후 일반공급(60%)·추첨(10%) — 신규 신청분/물량은 공고마다 달라 공고문 확인.
- 공급구조: 우선공급 30%(가점)→일반 60%→추첨 10%. (신혼희망타운 자체 가점표는 별도 — 미인코딩.)
- 출처: LH 신혼희망타운 안내 / 마이홈 / 뉴:홈. 값은 연 1회 갱신 필요.

### 6-4. UserProfile 추가 입력값 (분양·가점용)
§1 공통(생년월일·무주택·세대원수·소득·자산·자동차·거주지·해당지역연속거주·혼인신고일·자녀목록) 재사용 + 아래 추가.
| 필드 | 타입 | 용도 |
|---|---|---|
| 청약통장 가입일 | date | 가입기간(17점) + 1·2순위 |
| 청약통장 종류 | enum(종합저축/청약저축/예금/부금) | 민영(예치금)·공공(납입) 분기 |
| 예치금액 / 납입인정횟수·저축총액 | int | 민영 지역별 예치금 충족 / 공공 순위 |
| 부양가족 상세 | obj | 직계존속 3년↑ 동일등본 부양 여부, 미혼자녀(만30세미만) — 부양가족수(35점) 산정 |
| 무주택 기점 | date(파생) | 만30세 vs 혼인신고일 중 빠른 날 (무주택기간 32점) |
| 생애최초/신혼/다자녀/노부모 해당 | bool(파생가능) | 특공 매칭(자녀목록·혼인일·세대원으로 대부분 파생) |

> 대부분 임대용 프로필과 겹친다. **신규 입력은 청약통장(가입일·종류·예치/납입)뿐**, 나머지(무주택기점·부양가족수·특공해당)는 기존 입력에서 파생.
